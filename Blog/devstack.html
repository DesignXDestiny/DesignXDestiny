CSS Pre-Processors

May 2021
I am adding CSS preprocessing to my workflow. I am learning an important part of what is required for creating dynamic websites that are compatible with multiple browsers. Cascading Style Sheets and I have been well acquainted for years. The main CSS file for my website is about 900 lines of code. I have been diligent in my attention to detail skills, like finding the missing tag(s) and code breaking culprits. As many times as this style sheet validates, there are still unintentional cross-browser differences.

SASS & SCSS

Now I'm exploring SASS and SCSS to overcome the challenge of cross-browser compatibility checking, and some functionality in an efficient way. This page was styled using SASS. In SCSS, Erlenmeyer brackets and semi-colons are still used, similar to traditional CSS. I chose this over other preprocessors for online support. I appreciate it when other people have already asked, answered questions, and made tutorials.

Git

March 2021

It also works with my external server, IONOS where I keep my website hosted. The login process is very similar. My computer doesn't like for me to upload the same repo to both places, so I still end up doing it manually. The terminal is useful as a different way of looking through my files, the finder is pretty slow.
